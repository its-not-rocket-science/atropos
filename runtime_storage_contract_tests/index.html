Runtime storage backend contract tests

This project includes a backend contract suite that validates the shared AtroposStore behavior across all storage implementations.

Current backend adapters:

  • InMemoryStore
  • RedisStore

Contract tests live in:

  • tests/test_api_store_backends.py
  • adapter definitions: tests/contracts_store_adapters.py

Behavioral expectations covered

The suite verifies the core store-layer contract for:

  1. enqueue
  2. first enqueue inserts a queued job
  3. queue depth increments correctly
  4. get_job_status() returns queued metadata
  5. dedupe
  6. idempotency key deduplicates duplicate enqueue_job requests
  7. repeated scored-data request/group ingestion is deduplicated
  8. buffer
  9. ingestion transitions through buffered lifecycle metadata
  10. batch selection
  11. list_scored_data(..., limit=N) returns a bounded batch slice
  12. env registration
  13. explicit environment registration is idempotent
  14. registered environment inventory can be listed
  15. unknown environments return empty results
  16. ingested records are isolated per environment_id
  17. step/status updates
  18. group lifecycle reaches acknowledged
  19. lifecycle timestamps are set and monotonic
  20. durability semantics
  21. durable backends recover state across adapter-driven restart
  22. non-durable backends do not persist across restart

Adding a future backend to the contract suite

When a new store implementation is added, wire it into tests/contracts_store_adapters.py:

  1. Add a BackendAdapter factory that can:
  2. build a fresh store instance (build)
  3. build a restarted store against the same underlying persistence (restart)
  4. declare durability (durable=True/False)
  5. Add the adapter name to CONTRACT_BACKEND_NAMES and handle it in create_backend_adapter(...).
  6. Run the required checks:
ruff check .
ruff format --check .
pytest -m "not integration and not stress" --cov-report=xml:coverage.xml

If the backend is durable, restart assertions must pass with persisted state. If the backend is process-local/non-durable, restart assertions must show empty recovered state.
